The Problem

Why can't we just use new Thread()?

In a Jakarta EE server, creating raw threads is dangerous. The server has no control over them — no security context, no transaction propagation, no lifecycle management.

💀

Raw Threads

  • No security context
  • No JNDI access
  • Memory leaks
  • Uncontrolled lifecycle
✅

Jakarta Concurrency

  • Container-managed
  • Context propagation
  • Secure & monitored
  • Resource pooling
Section 01

What is Jakarta Concurrency?

📦

Part of Jakarta EE

A standardized API (formerly JSR 236) that provides concurrency utilities specifically designed for enterprise server applications.

🧵

Container-Managed

All threads are created and supervised by the application server. The container controls thread pools, lifecycle, and context propagation.

🔗

Context-Aware

Threads automatically inherit the security context, JNDI namespace, class loader, and other enterprise contexts from the calling thread.

💡 Think of it as java.util.concurrent — but redesigned for enterprise servers.

The 4 Core Managed Objects

⚡

ManagedExecutorService

Submit async tasks (Runnable / Callable)

⏰

ManagedScheduledExecutorService

Schedule tasks with delays or at fixed rates

🏭

ManagedThreadFactory

Create container-managed threads

🧩

ContextService

Capture & propagate contexts to threads

Section 02

ManagedExecutorService

📖 What is it?

A managed version of java.util.concurrent.ExecutorService that the container provides. It executes tasks asynchronously while preserving the enterprise context.

✓ Inject with @Resource
✓ Submit Runnable & Callable tasks
✓ Returns Future<T> for results
✓ Container handles thread lifecycle
OrderService.java 
@Stateless
public class OrderService {

  @Resource
  ManagedExecutorService executor;

  public void placeOrder(Order order) {
    // Save order synchronously
    orderRepo.save(order);

    // Fire-and-forget email
    executor.submit(() -> {
      emailService.send(
        order.getEmail(),
        "Order Confirmed! 🎉"
      );
    });
  }
}

Section 03

ManagedScheduledExecutorService

📖 What is it?

Extends ManagedExecutorService to add scheduling capabilities. You can run tasks after a delay or repeat them at fixed intervals — all container-managed.

✓ schedule() — run once after delay
✓ scheduleAtFixedRate() — repeat every N units
✓ scheduleWithFixedDelay() — delay between runs
✓ Returns ScheduledFuture<T>
HealthCheckService.java 
@Singleton
@Startup
public class HealthCheckService {

  @Resource
  ManagedScheduledExecutorService
      scheduler;

  @PostConstruct
  public void init() {
    scheduler.scheduleAtFixedRate(
      () -> {
        boolean ok = db.ping();
        log.info("DB alive: " + ok);
      },
      0, 30, TimeUnit.SECONDS
    );
  }
}

Section 04

Asynchronous Tasks

🔥 Fire-and-Forget

Submit a task and don't wait for the result. Perfect for logging, emails, analytics.

executor.submit(() -> sendEmail());

📬 Future-Based

Submit a task and get a Future<T> to check the result later.

Future<Report> f = executor.submit(() -> generateReport());
Report r = f.get(); // blocks until ready

🔗 CompletableFuture

Chain multiple async operations in a pipeline.

executor.supplyAsync(() -> fetchData())
.thenApply(data -> transform(data))
.thenAccept(result -> save(result));

Section 05

🔒 Secure Thread Management

What Gets Propagated?

Calling Thread
🔐 Security 📂 JNDI 📦 ClassLoader 💰 Transaction*
➡️ Auto-propagated ➡️
Managed Thread
🔐 Security ✓ 📂 JNDI ✓ 📦 ClassLoader ✓ 💰 New Transaction
⚠️ Note: Transactions do NOT propagate — each async task gets its own transaction context. This is by design for data integrity!

🧩 ContextService — Manual Context Capture

Sometimes you need to explicitly capture context — for example, when passing lambdas to non-managed executors:

How it works

  1. Capture — Create a contextual proxy of your task
  2. Pass — Hand the proxy to any executor
  3. Execute — The proxy restores context before running
ContextExample.java 
@Resource
ContextService ctxService;

public void process() {
  Runnable task = () -> {
    // needs security context!
    auditLog.record(
      "Action by: " +
      ctx.getCallerPrincipal()
    );
  };

  // Wrap with context
  Runnable ctxTask = ctxService
    .contextualRunnable(task);

  someExternalExecutor
    .submit(ctxTask); // ✅ Safe!
}

🏭 ManagedThreadFactory

Need to create individual threads? Use the factory — it creates container-aware threads:

LongPollingService.java 
@Resource
ManagedThreadFactory threadFactory;

public void startLongPolling() {
    Thread pollingThread = threadFactory.newThread(() -> {
        while (!Thread.currentThread().isInterrupted()) {
            List<Event> events = externalApi.poll();  // Long-polling
            events.forEach(e -> eventBus.publish(e));
        }
    });
    pollingThread.setName("external-api-poller");
    pollingThread.start();
}

// ✅ Thread has JNDI, security context, and classloader
// ✅ Container can monitor and shut down this thread
Section 06

Real-World Use Cases

📦

Batch Processing

Process thousands of invoices at the end of the month. Split work across multiple managed threads for parallel execution.

invoices.parallelStream().forEach(inv ->
  executor.submit(() -> process(inv)));
🔔

Notifications

Send push notifications, SMS, or emails without blocking the user's HTTP request thread.

executor.submit(() ->
  pushService.send(userId, message));
🔄

Long-Running Tasks

PDF generation, video transcoding, report building — offload to background threads and notify when done.

Future<File> pdf = executor.submit(
  () -> pdfEngine.generate(data));

🛒 Full Example: E-Commerce Order Pipeline

Combining everything — a real-world e-commerce backend using Jakarta Concurrency:

ECommerceOrderPipeline.java 
@Stateless
public class ECommerceOrderPipeline {

    @Resource ManagedExecutorService executor;
    @Resource ManagedScheduledExecutorService scheduler;

    public void processOrder(Order order) {
        // 1. Save order (sync)
        orderRepo.save(order);

        // 2. Fire parallel async tasks
        Future<Invoice> invoice = executor.submit(() -> billing.createInvoice(order));
        executor.submit(() -> emailService.sendConfirmation(order));
        executor.submit(() -> inventory.reserve(order.getItems()));
        executor.submit(() -> analytics.trackPurchase(order));

        // 3. Schedule delivery status check in 2 hours
        scheduler.schedule(
            () -> deliveryService.checkStatus(order.getId()),
            2, TimeUnit.HOURS
        );
    }
}
🎯 One method, multiple concerns — all handled safely and efficiently by the container!

Key Takeaways

🚫 Never use raw new Thread() in Jakarta EE
⚡ Use ManagedExecutorService for async tasks
⏰ Use ManagedScheduledExecutorService for timed tasks
🔒 Context (security, JNDI) is auto-propagated
🎯 Perfect for batch, notifications & long processes
